The Companies API gives you direct access to Specter's company intelligence. Each profile aggregates structured data across funding rounds, hiring trends, web traffic, revenue estimates, social traction, and proprietary growth signals.

## Get a company profile

**`GET /api/v1/companies/{companyId}`**

Retrieve the full enriched profile for a single company by its Specter ID.

<Note>
  This endpoint uses 1 credit per request. See [Plans & Pricing](/pricing) for the full credit model.
</Note>

### Path parameters

<ParamField path="companyId" type="string" required>
  The unique Specter ID for the company — a 24-character identifier such as `1a2b3c4d5e6g1a2b3c4d5e6g`. You can obtain it from a search result or a stored record.
</ParamField>

### Example request

```bash
curl --request GET \
  --url "https://app.tryspecter.com/api/v1/companies/1a2b3c4d5e6g1a2b3c4d5e6g" \
  --header "X-API-Key: YOUR_API_KEY"
```

### Response fields

<ResponseField name="id" type="string">
  Specter's unique 24-character identifier for the company. Use it when querying related endpoints such as transactions or signals.
</ResponseField>

<ResponseField name="organization_name" type="string">
  The company's main public name.
</ResponseField>

<ResponseField name="description" type="string">
  A summary of what the company does and who it is.
</ResponseField>

<ResponseField name="industry" type="array">
  Specter's industry classification — the market the company serves. One industry per company, with up to three levels. Replaces the deprecated `industries` field.
</ResponseField>

<ResponseField name="tech_verticals" type="array">
  The technology domains the company operates in (e.g. AI & Machine Learning, FinTech). Up to five verticals, each with two levels, returned as an array of arrays.
</ResponseField>

<ResponseField name="operating_status" type="string">
  Current operational status. One of: `active`, `acquired`, `closed`, or `ipo`.
</ResponseField>

<ResponseField name="growth_stage" type="string">
  Specter's maturity classification. One of: `bootstrapped`, `seed_stage`, `early_stage`, `growth_stage`, `late_stage`, or `exit_stage`.
</ResponseField>

<ResponseField name="employee_count" type="integer">
  The most recent headcount estimate.
</ResponseField>

<ResponseField name="employee_count_range" type="string">
  Headcount as a band. One of: `1-10`, `11-50`, `51-200`, `201-500`, `501-1000`, `1001-5000`, `5001-10000`, `10001+`.
</ResponseField>

<ResponseField name="revenue_estimate_usd" type="integer">
  Specter's modelled annual revenue estimate in USD.
</ResponseField>

<ResponseField name="highlights" type="array">
  Signal labels flagging notable activity, such as `strong_headcount_growth`, `top_tier_investors`, `headcount_surge`, and `strong_web_traffic_growth`. A `new_highlights` field lists those new this month.
</ResponseField>

<ResponseField name="founded_year" type="integer">
  The year the company was founded.
</ResponseField>

<ResponseField name="funding" type="object">
  Aggregated funding data for the company.
  <Expandable title="funding fields">
    <ResponseField name="total_funding_usd" type="integer">
      Total capital raised across all rounds, in USD.
    </ResponseField>
    <ResponseField name="last_funding_usd" type="integer">
      Amount raised in the most recent round, in USD.
    </ResponseField>
    <ResponseField name="last_funding_date" type="string">
      Date of the most recent funding event (YYYY-MM-DD).
    </ResponseField>
    <ResponseField name="last_funding_type" type="string">
      Round type of the most recent event, e.g. `seed`, `series_a`, `series_b`, `private_equity`.
    </ResponseField>
    <ResponseField name="round_count" type="integer">
      Number of tracked funding rounds. Per-round detail is available in `round_details`.
    </ResponseField>
    <ResponseField name="post_money_valuation_usd" type="integer">
      Post-money valuation in USD from the most recent round where disclosed.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="hq" type="object">
  Headquarters location.
  <Expandable title="hq fields">
    <ResponseField name="city" type="string">City of the headquarters.</ResponseField>
    <ResponseField name="state" type="string">State or province. Outside the US this relates to the country and region.</ResponseField>
    <ResponseField name="country" type="string">Country of the headquarters.</ResponseField>
    <ResponseField name="continent" type="string">The continent the HQ is in. Use this instead of the deprecated `region` field.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="web" type="object">
  Website traffic and engagement metrics.
  <Expandable title="web fields">
    <ResponseField name="visits" type="integer">Estimated monthly website visits.</ResponseField>
    <ResponseField name="popularity_rank" type="integer">Global website popularity rank. Lower values indicate higher traffic.</ResponseField>
    <ResponseField name="top_country" type="string">The country sending the most traffic.</ResponseField>
    <ResponseField name="bounce_rate" type="number">Bounce rate as a percentage (e.g. `41` = 41%).</ResponseField>
    <ResponseField name="pages_per_visit" type="number">Average pages viewed per session.</ResponseField>
    <ResponseField name="session_duration" type="integer">Average session duration in seconds.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="socials" type="object">
  Social profiles and follower counts. Each network is an object with a `url` and, where available, a `follower_count`.
  <Expandable title="socials fields">
    <ResponseField name="twitter" type="object">Twitter/X `url` and `follower_count`.</ResponseField>
    <ResponseField name="linkedin" type="object">LinkedIn `url` and `follower_count`.</ResponseField>
    <ResponseField name="instagram" type="object">Instagram `url`, `follower_count`, and `following_count`.</ResponseField>
    <ResponseField name="facebook, itunes, googleplay, crunchbase, pitchbook" type="object">Further profiles, each with a `url` (and app or review fields where relevant).</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="traction_metrics" type="object">
  Time-series growth metrics. Each metric — `web_visits`, `popularity_rank`, `employee_count`, `linkedin_followers`, `twitter_followers`, `instagram_followers`, app and review metrics, and more — holds a `latest` value plus change objects for `1mo`, `2mo`, `3mo`, `6mo`, `12mo`, and `24mo`.
  <Expandable title="how each change object works">
    <ResponseField name="latest" type="number">The current value of the metric.</ResponseField>
    <ResponseField name="value" type="number">The metric's value at that earlier period.</ResponseField>
    <ResponseField name="change" type="number">Absolute difference between the latest value and that period.</ResponseField>
    <ResponseField name="percentage" type="number">Percentage change relative to that period.</ResponseField>
  </Expandable>
</ResponseField>

## Error codes

| Status | Meaning |
|--------|---------|
| `200` | Success — company profile returned |
| `401` | Missing or invalid API key |
| `403` | Your plan doesn't include access to this endpoint |
| `404` | No company found for the provided ID |
| `429` | Rate limited, or out of credits (`OUT_OF_CREDITS`). Check `X-RateLimit-Reset` or `X-CreditLimit-Reset` |

See the [API overview](/api/overview) for the full list of status codes.
