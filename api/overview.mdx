The Specter API gives you programmatic access to the same private markets intelligence that powers the Specter platform. Use it to build enrichment pipelines, pull company and people data into your own tools, or automate deal sourcing.

Every response is JSON, and the resource you asked for is returned directly — no envelope to unwrap.

## Base URL

All requests go to:

```
https://app.tryspecter.com/api/v1
```

Append the resource path to this base URL when building a request.

## Core endpoints

A few of the most-used endpoints are below. The full list of 73 endpoints lives in the sidebar under each resource.

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/companies` | Enrich a company by domain, LinkedIn or Crunchbase URL |
| `GET` | `/companies/search` | Find a company by name |
| `GET` | `/companies/{companyId}` | Get a full company profile by ID |
| `POST` | `/people` | Enrich a person by LinkedIn URL or email |
| `GET` | `/investors` | Look up an investor by website or name |
| `GET` | `/talent/{talentSignalId}` | Get a talent signal by ID |
| `GET` | `/revenue-signals` | List revenue signals |
| `GET` | `/searches/companies/{searchId}/results` | Pull the results of a saved company search |
| `GET` | `/lists/companies/{listId}/results` | Pull the companies in a list |

## Response format

Successful responses return the resource directly. A single record comes back as a JSON object:

```json
{
  "id": "1a2b3c4d5e6g1a2b3c4d5e6g",
  "organization_name": "Hedgehog Analytics",
  "domain": "hedgehog_analytics.com",
  "hq": { "city": "London", "country": "United Kingdom" },
  "employee_count": 39,
  "funding": {
    "total_funding_usd": 16600000,
    "last_funding_type": "series_a"
  },
  "last_updated": "2026-01-01"
}
```

Search and list endpoints return an array of records. Pagination is handled through response headers, not a wrapper object — `X-Total-Count`, `X-Page`, `X-Page-Size`, `X-Total-Pages`, and a `Link` header for first/prev/next/last.

## Error codes

The API uses standard HTTP status codes. Every error body carries an `errorCode` and a `message`.

| Code | Meaning |
|------|---------|
| `400` | Bad request — a field is missing or malformed. Doesn't cost credits, but counts against your rate limit. |
| `401` | Unauthorized — your key is missing or invalid. |
| `402` | `OUT_OF_CREDITS` — your team has used its credits for the period. Check `X-CreditLimit-Reset` for when they renew. |
| `403` | Forbidden — your plan doesn't include this endpoint. |
| `404` | Not found — the resource doesn't exist. |
| `422` | Validation error — the `message` explains what to fix. |
| `429` | `RATE_LIMITED` — too many requests per second. Check `X-RateLimit-Reset`. |
| `50x` | Server error on our side. Won't cost credits; you may have been rate limited. |

You can make up to 15 requests per second per key. Search and enrich calls draw on your credit allowance — search by the result, enrich by the record returned.
