Specter uses API key authentication. Every request must include your key in the `X-API-Key` header — requests without a valid key are rejected with `401 Unauthorized`. There are no OAuth flows or token exchanges to manage; your key is all you need.

Your firm has one team-level key. Everyone on the team works from it, and rate and credit limits apply at the workspace level.

## Getting your API key

<Steps>
  <Step title="Get access">
    Specter is approval-gated. If your firm isn't set up yet, request access at [tryspecter.com/contact](https://www.tryspecter.com/contact). Approved firms get an invitation to the web app.
  </Step>
  <Step title="Sign in">
    Sign in to the [Specter dashboard](https://app.tryspecter.com).
  </Step>
  <Step title="Open the API Console">
    Go to **Settings → API Console** at [app.tryspecter.com/settings/api-console](https://app.tryspecter.com/settings/api-console).
  </Step>
  <Step title="Copy your key">
    Copy your team's key and store it somewhere secure. If there's no key yet, click **Request API Access** — requests are reviewed and approved the same day.
  </Step>
</Steps>

## Making authenticated requests

Pass your key in the `X-API-Key` header on every request:

```bash
curl --request GET \
  --url https://app.tryspecter.com/api/v1/companies/1a2b3c4d5e6g1a2b3c4d5e6g \
  --header 'X-API-Key: YOUR_API_KEY'
```

The header works the same way across every HTTP client:

<CodeGroup>

```bash curl
curl --request GET \
  --url https://app.tryspecter.com/api/v1/companies/1a2b3c4d5e6g1a2b3c4d5e6g \
  --header 'X-API-Key: YOUR_API_KEY'
```

```python Python
import requests

headers = {"X-API-Key": "YOUR_API_KEY"}
response = requests.get(
    "https://app.tryspecter.com/api/v1/companies/1a2b3c4d5e6g1a2b3c4d5e6g",
    headers=headers
)

data = response.json()
print(data)
```

```javascript JavaScript
const response = await fetch(
  'https://app.tryspecter.com/api/v1/companies/1a2b3c4d5e6g1a2b3c4d5e6g',
  { headers: { 'X-API-Key': 'YOUR_API_KEY' } }
);

const data = await response.json();
console.log(data);
```

```go Go
req, _ := http.NewRequest("GET", "https://app.tryspecter.com/api/v1/companies/1a2b3c4d5e6g1a2b3c4d5e6g", nil)
req.Header.Set("X-API-Key", "YOUR_API_KEY")
client := &http.Client{}
resp, err := client.Do(req)
```

</CodeGroup>

## Request header reference

<ParamField header="X-API-Key" type="string" required>
  Your team's Specter API key, from **Settings → API Console** in the [dashboard](https://app.tryspecter.com/settings/api-console). Every request must include this header.
</ParamField>

## Authentication errors

| Code | Cause | How to fix |
|------|-------|------------|
| `401` | Missing or invalid key | Confirm you're sending the `X-API-Key` header (not `Authorization: Bearer`) and that the key is correct and active. |
| `403` | Endpoint not on your plan | Your plan doesn't include this endpoint — [contact support](mailto:azher@tryspecter.com) to upgrade. |
| `429` | Rate limited | You've exceeded 15 requests per second. Back off and retry, checking the `X-RateLimit-Reset` header. |

For the full list of status codes, including credit limits, see the [API overview](/api/overview).

<Warning>
  Treat your key like a password. Anyone who has it can reach your firm's data.

  - Never put it in client-side code, browser JavaScript, or a public repository.
  - Store it in an environment variable (e.g. `SPECTER_API_KEY`), not hardcoded in source.
  - If it's exposed, rotate it from the [API Console](https://app.tryspecter.com/settings/api-console) and the old key stops working.
</Warning>
