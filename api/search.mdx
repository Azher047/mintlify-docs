The Search API lets you query Specter's full intelligence graph using natural language, structured filters, or both together. You can describe what you're looking for in plain English — "growth stage AI companies in Europe with recent headcount surges" — and Specter will return ranked, enriched results. Pair natural language with filters for precision at scale, or use filters alone when you already know the exact parameters.

<Tip>
  Natural language queries are especially powerful for exploratory research. Specter's search model understands business context — you can use terms like "stealth mode infrastructure startups" or "Series A ready healthtech companies with top-tier backers" and get semantically relevant results without needing to map every concept to a specific filter.
</Tip>

## Search Companies

**`POST /api/v1/companies/search`**

Search across Specter's full company database using a natural language query, a structured filter set, or a combination of both. Results are returned as ranked company summaries.

### Request Body

<ParamField body="query" type="string">
  A natural language description of the companies you're looking for. Example: `"growth stage AI video companies in Europe"`. Optional when using filters alone.
</ParamField>

<ParamField body="filters" type="object">
  A structured filter object to narrow results. All sub-fields are optional and combinable.
  <Expandable title="filter fields">
    <ParamField body="industries" type="array">
      List of industry tags to match against. Example: `["Artificial Intelligence", "Healthcare"]`.
    </ParamField>
    <ParamField body="growth_stage" type="array">
      Filter by funding and maturity stage. Accepted values: `early_stage`, `growth_stage`, `late_stage`, `pre_launch`.
    </ParamField>
    <ParamField body="hq_country" type="string">
      Filter to companies headquartered in a specific country. Example: `"Germany"`.
    </ParamField>
    <ParamField body="employee_count_min" type="integer">
      Return only companies with at least this many employees.
    </ParamField>
    <ParamField body="employee_count_max" type="integer">
      Return only companies with at most this many employees.
    </ParamField>
    <ParamField body="founded_year_min" type="integer">
      Return only companies founded in or after this year.
    </ParamField>
    <ParamField body="has_funding" type="boolean">
      When `true`, return only companies with at least one recorded funding round.
    </ParamField>
    <ParamField body="highlights" type="array">
      Filter to companies with one or more of these signal labels. Example: `["headcount_surge", "top_tier_investors"]`.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="limit" type="integer">
  Number of results to return. Defaults to `20`. Maximum is `100`.
</ParamField>

<ParamField body="offset" type="integer">
  Number of results to skip for pagination. Defaults to `0`.
</ParamField>

### Example Request

```bash
curl --request POST \
  --url "https://app.tryspecter.com/api/v1/companies/search" \
  --header "X-API-Key: YOUR_API_KEY" \
  --header "Content-Type: application/json" \
  --data '{
    "query": "growth stage AI video companies in Europe",
    "filters": {
      "growth_stage": ["growth_stage"],
      "hq_country": "Germany",
      "has_funding": true,
      "highlights": ["top_tier_investors"]
    },
    "limit": 3,
    "offset": 0
  }'
```

## Search People

**`POST /api/v1/people/search`**

Search across Specter's people database to find founders, executives, and operators matching your criteria. Combine natural language with structured filters to identify individuals by background, highlights, title, or location.

### Request Body

<ParamField body="query" type="string">
  A natural language description of the people you're looking for. Example: `"serial founders with prior exits now building in AI"`. Optional when using filters alone.
</ParamField>

<ParamField body="filters" type="object">
  A structured filter object to narrow results. All sub-fields are optional and combinable.
  <Expandable title="filter fields">
    <ParamField body="current_company_id" type="string">
      Return only people currently employed at this company. Use a Specter company ID.
    </ParamField>
    <ParamField body="highlights" type="array">
      Filter to people with one or more of these signal labels. Accepted values: `Prior Exit`, `Serial Founder`, `VC-Backed Founder`, `Top University`.
    </ParamField>
    <ParamField body="current_title_keywords" type="array">
      Return only people whose current job title contains one or more of these keywords. Example: `["CTO", "VP Engineering"]`.
    </ParamField>
    <ParamField body="location_country" type="string">
      Filter to people based in a specific country. Example: `"United Kingdom"`.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="limit" type="integer">
  Number of results to return. Defaults to `20`. Maximum is `100`.
</ParamField>

<ParamField body="offset" type="integer">
  Number of results to skip for pagination. Defaults to `0`.
</ParamField>

### Example Request

```bash
curl --request POST \
  --url "https://app.tryspecter.com/api/v1/people/search" \
  --header "X-API-Key: YOUR_API_KEY" \
  --header "Content-Type: application/json" \
  --data '{
    "query": "serial founders with prior exits now building in AI",
    "filters": {
      "highlights": ["Prior Exit", "Serial Founder"],
      "location_country": "United Kingdom"
    },
    "limit": 3,
    "offset": 0
  }'
```

## Error Codes

| Status | Meaning |
|--------|---------|
| `200` | Success — search results returned |
| `401` | Missing or invalid API key |
| `403` | Your plan does not include access to this endpoint |
| `404` | No results found matching the specified query or filters |
| `429` | Rate limit exceeded — slow down your request cadence |
